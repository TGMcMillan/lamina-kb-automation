You are helping maintain Lamina’s knowledge base.

Look in the /kb-source/ folder and identify any files that contain:
- new product information
- updated workflows
- release notes
- corrected process steps
- terminology changes

Then do the following:

1. Summarize the meaningful changes in plain English.
2. Create or update a markdown file in /kb-summaries/daily-summary.md.
3. If a change should affect a KB article, note it clearly.
4. Do not invent product behavior. If something is unclear, mark it as “Needs Review.”
5. Keep the writing concise and structured with headings and bullets.

At the top of the summary include:
- Date
- Files reviewed
- Articles likely affected
- Open questions
